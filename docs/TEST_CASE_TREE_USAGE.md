# 测试用例分类树使用说明

## 📋 概述

测试套件创建/编辑页面现在支持**多层级分类树**展示，与API仓库的分类树保持一致的视觉效果和交互体验。

## 🎯 功能特性

### 1. **多层级分类支持**
- 支持通过 `/` 分隔的 category 字段自动构建多层级树形结构
- 自动识别和展示层级关系
- 层级缩进显示，视觉清晰

### 2. **展开/折叠交互**
- 点击节点左侧的 `>` 图标展开/折叠子节点
- 点击节点本身可选中该分类并筛选测试用例
- 再次点击已选中的节点可取消选择

### 3. **实时统计**
- 每个节点右侧显示该分类下的用例数量
- 顶部显示总用例数和分类统计
- 支持"未分类"用例的独立显示

## 📝 测试用例分类格式

### 单层分类
```json
{
  "name": "登录接口测试",
  "category": "用户管理"
}
```

显示效果：
```
└─ 用户管理 (1)
```

### 多层分类（推荐）
```json
{
  "name": "登录接口测试",
  "category": "用户管理/登录"
}
```

显示效果：
```
└─ 用户管理
   └─ 登录 (1)
```

### 三层分类示例
```json
{
  "name": "创建订单测试",
  "category": "ITOMV/订单管理/创建订单"
}
```

显示效果：
```
└─ ITOMV
   └─ 订单管理
      └─ 创建订单 (1)
```

## 💡 使用建议

### 1. **命名规范**
- 使用清晰的层级命名：`平台/模块/功能`
- 避免使用特殊字符（除了 `/`）
- 保持命名简洁明了

### 2. **层级设计**
- **第一层**：平台或系统名称（如：ITOMV、测试平台）
- **第二层**：功能模块（如：用户管理、订单管理）
- **第三层**：具体功能（如：登录、创建订单）

### 3. **分类示例**
```
✅ 推荐：
- "ITOMV/用户管理/登录"
- "测试平台/API管理/创建API"
- "订单系统/订单管理/查询订单"

❌ 不推荐：
- "ITOMV用户管理登录"（缺少层级）
- "ITOMV//用户管理"（空层级）
- "ITOMV/用户管理/"（末尾多余斜杠）
```

## 🔧 技术实现

### 组件结构
```
components/test-suites/
├── TestCaseTree.tsx        # 分类树组件
├── TestCaseSelector.tsx    # 用例选择器（集成分类树）
├── PlatformSettingsForm.tsx
└── ScheduleConfig.tsx
```

### 核心功能
1. **自动解析**：根据 `/` 分隔符自动构建树形结构
2. **展开状态管理**：使用 `Set<string>` 跟踪展开的节点
3. **计数统计**：实时计算每个节点的用例数量
4. **筛选联动**：选择节点后自动筛选右侧用例列表

### 数据流
```
测试用例数据 (category字段)
    ↓
解析并构建树形结构
    ↓
渲染分类树（支持展开/折叠）
    ↓
用户选择分类
    ↓
筛选并显示对应的测试用例
```

## 🎯 最佳实践

### 创建测试用例时
1. 在用例编排页面创建测试用例
2. 在"分类"字段填写多层级路径（用 `/` 分隔）
3. 示例：`ITOMV/用户管理/登录`

### 在测试套件中使用
1. 点击"新建测试套件"或"编辑测试套件"
2. 在"选择测试用例"卡片中，左侧会显示分类树
3. 点击分类节点查看该分类下的用例
4. 勾选需要的用例，添加到测试套件

## 🐛 常见问题

### Q: 为什么我的分类没有显示层级？
**A:** 确保使用 `/` 作为分隔符。例如：`平台/模块/功能`

### Q: 如何批量修改用例的分类？
**A:** 目前需要在用例编排页面逐个编辑用例的分类字段。

### Q: 未分类的用例会显示吗？
**A:** 会的！没有 category 字段的用例会显示在"未分类"节点中。

### Q: 分类树的展开状态会保存吗？
**A:** 当前页面刷新后展开状态会重置。如需持久化可以使用 localStorage。
